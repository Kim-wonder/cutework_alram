# [workOut_alram] FRD

## 0. 문서 개요
- **프로젝트명** : 운동알람 (workOut_alram)
- **목적** : 사용자가 사전에 설정한 알람 시간에 알림을 통해 유튜브 운동 영상을 즉시 실행하도록 지원
- **범위** : MVP (구현 완료 버전, PWA v12 기준)
- **대상 플랫폼** : Android 웹앱 (PWA) — 모바일 브라우저·홈 화면 설치
- **전제**
  - 앱 내부 영상 재생 없음
  - 유튜브 공식 앱 또는 웹 브라우저로만 실행
  - 서버·로그인 없음 (모든 데이터 로컬 저장)

## 1. 용어정리

| 용어 | 정의 |
|---|---|
| 알람 | 사용자가 앱에서 알람명·요일·시간·영상을 설정한 운동 일정 데이터 (매주 반복) |
| 알림 | 알람에 의해 OS에서 실제로 발생하는 Notification 메시지 |
| 알람 화면 | 알람 도래 시 앱 내에 표시되는 전체 화면 다이얼로그 (재생/스누즈/종료) |
| 영상 실행 | 유튜브 딥링크 또는 웹 링크를 통해 외부 앱/브라우저에서 영상 재생 |
| 재알림(스누즈) | 알림 미클릭 시 5분 간격으로 반복되는 추가 알림 |

## 3. 기능 상세 정의

### F-01. 운동 알람 생성 (3단계 플로우)
**설명** — 사용자가 3단계에 걸쳐 알람을 생성한다.

**STEP 1/3 — 시간·요일**
- 입력값 : 요일(복수 선택 가능), 시간(시:분)
- 선택한 요일마다 매주 반복 실행됨을 안내 문구로 노출
- 필수값 미입력 시 오류 안내 및 다음 단계 이동 불가

**STEP 2/3 — 유튜브 링크** (→ F-02, F-03)

**STEP 3/3 — 알람명**
- 입력값 : 알람명 (필수, 최대 30자)
- 조회된 영상 제목을 기본값으로 제안하며 수정 가능
- 미입력 시 "알람명을 입력해야 등록할 수 있어요" 오류 노출, 등록 불가

**결과** — 알람이 활성화 상태 + 고유 색상(F-13)으로 로컬 저장됨

---

### F-02. 유튜브 영상 링크 등록
- **입력값** : 유튜브 URL (watch, youtu.be, shorts, embed, live 허용)
- **처리 로직** : URL에서 videoId(11자) 추출, 실패 시 "유효한 유튜브 링크가 아니에요" 오류 안내
- **결과** : 유효한 유튜브 영상만 등록 가능

---

### F-03. 유튜브 영상 메타 정보 조회
- **조회 항목** : 영상 제목, 채널명, 썸네일 이미지
- **처리 방식** : YouTube oEmbed API → 실패 시 noembed 폴백
- **예외 처리**
  - 조회 실패 시 "영상 정보를 불러올 수 없습니다" 메시지 노출 (등록은 가능)
  - 제목이 비어있는 알람은 앱 실행 시·온라인 복귀 시 자동 재조회하여 보정 저장

---

### F-04. 로컬 알림 발송 (매주 반복)
- **알림 조건** : 알람이 활성화 상태, 알림 권한 허용 상태
- **반복 규칙** : 선택된 요일·시간에 매주 반복 발송 (별도 만료 없음)
- **발송 방식** : Service Worker 알림(백그라운드 탭 대응) 우선, 불가 시 페이지 Notification.
  동시에 인앱 알람 화면 표시 + 진동
- **알림 메시지** : "운동할 시간이에요 💪 지금 시작해볼까요?"
- **중복 방지** : 동일 알람·동일 분 내 중복 발송 차단

---

### F-05. 알림 클릭 시 유튜브 영상 실행 (핵심 기능)
**처리 순서**
1. Android : intent 딥링크 실행 시도
   `intent://www.youtube.com/watch?v={videoId}#Intent;scheme=https;package=com.google.android.youtube;S.browser_fallback_url={webUrl};end`
2. 유튜브 앱 미설치·실행 실패 시 웹 URL(`https://www.youtube.com/watch?v={videoId}`) 실행
3. 알림 클릭 시 Service Worker가 예약된 재알림 중단 신호를 앱에 전달

**제약** — 앱 내부 영상 재생 없음, 백그라운드 자동 재생 없음

---

### F-06. 재알림 기능 + 종료
- **조건** : 최초 알림 이후 사용자 액션 없음
- **동작** : 5분 간격 재알림, 최대 3회
- **알람 화면 액션**
  - [▶ 영상 재생하기] : 영상 실행 + 재알림 즉시 중단
  - [5분 뒤 다시 알림] : 화면 닫기, 재알림 유지
  - [종료] : 예약된 재알림 전체 취소 (다음 정기 알람까지 미발송)
- **중단 조건** : 알림 클릭, 알람 비활성화, 알람 수정·삭제 시 즉시 중단

---

### F-07. 영상 실행 실패 처리
- **실패 케이스** : 네트워크 연결 불가, 영상 삭제/비공개, 외부 앱 실행 실패
- **처리 방식** : 오류 메시지 노출 + [다시 시도] 버튼 제공
- **메시지** : "네트워크 연결이 불안정합니다. 인터넷 연결을 확인하고 재실행 해주세요."

---

### F-08. 알림 권한 관리
- 헤더에 권한 상태 배지 상시 노출 (요청 전 / 허용됨 / 차단됨)
- 알람 설정 시 권한 미허용 상태 감지 → 권한 요청
- 거부 상태에서는 브라우저 설정 이동 안내
- **안내 문구** : "운동 알림을 받으려면 알림 권한이 필요해요."

---

### F-09. 다음 알림 안내 (NEXT WORKOUT 배너)
- 활성 알람 중 현재 시간 기준 가장 가까운 다음 알람 계산
- **48시간 이내** : 실시간 카운트다운(시:분:초) + 캡슐 진행바 + "다음 운동 알림은 오늘 19:00에
  예정되어 있어요" 안내
- **48시간 초과** : 카운트다운 대신 알람명(대) + "○요일 HH:MM · 매주 반복" 표기
- 활성 알람이 없으면 빈 상태 안내 노출

---

### F-10. 알람 수정
- 알람 카드 [수정] 버튼 → 기존 값이 채워진 3단계 플로우 진입
- STEP 2에서 기존 링크·미리보기 즉시 노출, 링크 변경 시 재조회 필요
- [수정 완료] 시 알람 갱신, 진행 중이던 재알림 초기화, 고유 색상은 유지

---

### F-11. 알람 삭제 (확인 모달)
- [삭제] 클릭 → "알람을 삭제하시겠습니까?" 확인 모달 노출
- [확인] : 알람·재알림 삭제 / [취소] : 유지

---

### F-12. 요일별 알람 보기 (주간 스트립)
- 상단에 일~토 날짜 스트립 노출, 오늘 날짜는 링으로 표시
- **앱 시작 시 오늘 날짜가 자동 선택**되어 오늘의 알람만 노출 ("오늘의 알람 · N개")
- 날짜 클릭 : 해당 요일에 실행되는 알람 필터 / 같은 날짜 재클릭 : 전체 보기
- 해당 요일 알람이 없으면 빈 상태 안내 노출

---

### F-13. 알람 고유 색상 배정
- 8가지 파스텔(핑크→옐로→민트→블루→라일락→피치→로즈→라임)이 **생성 순서대로 순환 배정**
- 순번 카운터는 로컬 저장되어 삭제와 무관하게 이어짐
- 배정된 색상은 알람에 저장되어 정렬·필터·수정과 무관하게 유지

---

### F-14. 썸네일 대체 이미지
- 유튜브 썸네일 로딩 실패 시 해상도 순차 재시도 (hqdefault → mqdefault → default)
- 최종 실패 시 앱 내장 대체 일러스트 표시 (base64 내장 + `fallback-thumb.jpg`, 오프라인 대응)

---

### F-15. PWA 설치 및 오프라인 실행
- Web App Manifest : 앱 이름·아이콘(192/512)·standalone·세로 모드
- Service Worker : 앱 셸(HTML·manifest·아이콘·대체 이미지) 캐시 우선 제공, 오프라인 실행 지원
- 알림 클릭 처리(`notificationclick`) : 유튜브 실행 + 재알림 중단 메시지 전파
- 캐시 버전 관리로 업데이트 배포

## 4. 데이터 정의

### Alarm
| 필드 | 설명 |
|---|---|
| alarmId | 고유 식별자 |
| name | 알람명 (사용자 입력, 최대 30자) |
| color | 고유 파스텔 색상 키 (8색 중 1) |
| daysOfWeek | 반복 요일 배열 (0=일 ~ 6=토) |
| time | HH:MM |
| enabled | 활성화 여부 |
| video | Video 객체 (내장) |

### Video
| 필드 | 설명 |
|---|---|
| videoId | 유튜브 영상 ID (11자) |
| youtubeUrl | 정규화된 watch URL |
| title | 영상 제목 (oEmbed 조회) |
| channelName | 채널명 |
| thumbnailUrl | 썸네일 URL |

### 기타
- colorSeq : 색상 순환 배정용 누적 순번 카운터

**저장 방식** — 로컬(localStorage), 서버 연동 및 로그인 없음

## 5. 정책 및 제약
- 본 앱은 유튜브 공식 앱 또는 웹을 통해서만 영상을 실행한다.
- 유튜브 콘텐츠를 저장·변형·광고 제거하지 않는다.
- 유튜브 정책 및 OS 알림 정책을 준수한다.
- 웹 플랫폼 특성상 알림은 앱(탭)이 실행 중이거나 백그라운드에 살아있을 때 발송된다.
