# [workOut_alram] PRD

- **프로젝트명** : 운동알람 (workOut_alram)
- **작성 기준** : MVP 구현 완료 버전 (PWA v12)
- **플랫폼** : Android 웹앱 (PWA) — 모바일 브라우저 및 홈 화면 설치 지원
- **상태** : 🟢 MVP 구현 완료

---

## 1. 제품 핵심 정의

### 배경 (Problem)
운동 의지는 있으나, 매번 유튜브 영상을 다시 검색하고 재생해야 하는 번거로움으로 인해
운동 시작까지 진입 장벽이 높아 홈트레이닝을 지속하지 못한다.

### 해결방안 (Solution)
지정된 시간에 알림을 제공하고, 사용자가 알림을 클릭하면 유튜브 앱 또는 브라우저를 통해
사전에 등록한 운동 영상을 즉시 재생할 수 있도록 한다.
모든 알람은 사용자가 선택한 요일에 **매주 반복** 실행되어 습관 형성을 돕는다.

### 성공지표 (Metric)
- **스케줄 운동 시작률** : 알람 발생 후 5분 이내 재생 세션 시작
  - 알림 이후 재생 버튼을 누른다는 것은 사용자가 운동 알람을 확인하고 유튜브 영상을
    실행했다는 의미로, 습관 형성에 긍정적인 영향을 주고 있다고 판단
- **보조지표** : 알림 권한 허용률, 알림 클릭률

## 2. 프로젝트 범위 설정

우선순위: `[p0]` 최우선/긴급 · `[p1]` 높음 · `[p2]` 보통

### ✅ In-Scope (구현 완료)

| 우선순위 | 기능 | 상태 |
|---|---|---|
| p0 | 지정된 시간에 로컬 알림 발송 (매주 요일 반복) | ✅ |
| p0 | 알림 클릭 시 유튜브 딥링크 실행 (앱 우선, 미설치 시 브라우저) | ✅ |
| p0 | 3단계 알람 등록 플로우 (시간·요일 → 링크 → 알람명) | ✅ |
| p0 | 유튜브 메타 정보(제목·채널·썸네일) 조회 및 미리보기 | ✅ |
| p0 | 알림 권한 관리 및 안내 | ✅ |
| p0 | PWA 설치 (홈 화면 추가, 오프라인 앱 셸) | ✅ |
| p1 | 재알림 (5분 간격, 최대 3회, 클릭 시 중단) | ✅ |
| p1 | 알람 화면 [종료] 버튼 — 이후 스누즈 완전 중단 | ✅ |
| p1 | 영상 실행 실패 시 재시도 안내 | ✅ |
| p1 | 알람 수정 (시간·요일·링크·알람명) | ✅ |
| p1 | 알람 삭제 전 확인 모달 | ✅ |
| p1 | 요일별 알람 필터 (시작 시 오늘 자동 선택) | ✅ |
| p1 | NEXT WORKOUT 배너 (48h 이내 카운트다운 / 초과 시 알람 정보) | ✅ |
| p2 | 알람 고유 파스텔 색상 (8색 생성 순서 순환 배정) | ✅ |
| p2 | 썸네일 로딩 실패 시 대체 일러스트 표시 | ✅ |

### ⛔️ Out-of-Scope (나중에 / 안함)
- 동영상 여러 개 등록 후 순차 실행
- 해빗트래커 ('재생' 버튼을 누른 날 기록)
- 앱 내부 영상 재생
- 유튜브 다운로드 / 광고 제거
- 백그라운드 자동 재생
- 서버 연동 / 로그인 / 계정 동기화

## 3. 사용자 경험 및 규칙

### 3-1. 핵심 흐름 (User Flow)

- **Step 0** : 앱 실행 시 오늘 날짜가 선택된 상태로 시작하며, 오늘 실행되는 알람 목록과
  NEXT WORKOUT 배너가 노출된다.
- **Step 1** : 사용자가 'SET WORK OUT' 버튼 클릭
- **Step 2** : (STEP 1/3) 운동 시간·요일(복수) 선택 후 [다음] — 선택한 요일마다 매주 반복됨을 안내
- **Step 3** : (STEP 2/3) 유튜브 영상 링크 입력 후 [등록]
- **Step 4** : (시스템) 링크의 영상 정보(썸네일·제목·채널명) 조회 및 미리보기 노출
- **Step 5** : (성공 시) [다음] 버튼 노출 → 클릭
- **Step 6** : (STEP 3/3) 알람명 입력(필수, 최대 30자, 영상 제목 기본 제안) 후 [운동알람 등록]
- **Step 7** : (시스템) 알람이 8색 파스텔 중 생성 순번의 고유 색상으로 목록에 등록
- **Step 8** : (시스템) 알람 시간이 되면 OS 알림 + 인앱 알람 화면 표시
- **Step 9** : 사용자가 알림 클릭 또는 [▶ 영상 재생하기] 선택
- **Step 10** : (시스템) 유튜브 딥링크 실행 시도, 불가 시 기본 브라우저로 유튜브 링크 실행

### 3-2. 알람 화면 액션 (알람 도래 시)
- **▶ 영상 재생하기** : 유튜브 실행 + 예약된 재알림 즉시 중단
- **5분 뒤 다시 알림** : 화면 닫기, 예약된 재알림 유지 (5분 간격 최대 3회)
- **종료** : 예약된 재알림 전체 취소, 다음 정기 알람까지 대기

### 3-3. 엣지 케이스 (Key Logic & Edge Cases)
- **영상이 삭제/비공개된 경우** : 유튜브 딥링크로 연결 (유튜브 측 안내에 위임)
- **네트워크 연결이 끊어진 경우** : "네트워크 연결이 불안정합니다. 인터넷 연결을 확인하고
  재실행 해주세요." 안내 + [다시 시도] 버튼 제공
- **유튜브 앱 비로그인 상태** : 비로그인 상태로 실행
- **알림 권한 미수락** : 알람 설정 시 감지하여 권한 요청, 거부 시 설정 이동 안내
- **메타 조회 실패** : "영상 정보를 불러올 수 없습니다" 안내, 앱 재실행·온라인 복귀 시 자동 재조회
- **썸네일 로딩 실패** : 해상도 순차 재시도(hq→mq→default) 후 내장 대체 일러스트 표시
- **다음 알람이 48시간 초과** : 카운트다운 대신 알람명·요일·시간 표기

### 3-4. 정책 (Policy)
- 본 앱은 유튜브 공식 앱 또는 웹을 통해 영상을 재생하며, 유튜브 콘텐츠를 저장·변형·광고
  제거하지 않는다.
- 유튜브 정책 및 OS 알림 정책을 준수한다.

## 4. 기술 검토

- **저장 데이터(로컬)** : 알람명, 영상 링크·메타, 알람 시간, 알람 요일, 활성 상태, 고유 색상,
  색상 순번 카운터
- 로그인 없음, 서버 연동 없음 (localStorage 기반)
- **PWA 구성** : Web App Manifest(홈 화면 설치) + Service Worker(앱 셸 캐시·오프라인 실행·알림
  클릭 처리)
- **제약** : 웹 플랫폼 특성상 알림은 앱(탭)이 실행 중이거나 백그라운드에 살아있을 때 발송됨.
  완전한 백그라운드 알람이 필요할 경우 네이티브 전환 검토
